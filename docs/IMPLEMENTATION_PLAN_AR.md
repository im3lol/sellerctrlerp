# خطة تنفيذ دورات المشتريات والمبيعات والمخزون (مرجع معتمد)

> هذه الوثيقة هي **المرجع الرسمي** لتنفيذ دورات المستندات في SellerCtrl.
> اعتُمدت بتاريخ 2026-06-20. أي تنفيذ لاحق يجب أن يطابقها.

## القواعد الحاكمة
1. إنشاء/حفظ أي مستند يجعله **Draft فقط**.
2. المستند `Draft` لا يؤثر على المخزون أو الحسابات أو أرصدة العملاء/الموردين.
3. التأكيد يدوي عبر زر **تأكيد** بعد المراجعة.
4. لا يُنشأ المستند التالي تلقائيًا؛ يظهر زر مثل **إنشاء إذن استلام**.
5. المستند المؤكد لا يُعدّل/يُحذف؛ التصحيح بالإلغاء المنضبط أو بمستند عكسي.
6. كل مستند يحمل روابط المستندات السابقة واللاحقة (`sourceDocumentId`).
7. دعم التنفيذ والفوترة والدفع الجزئي.
8. كل تأكيد داخل **Database Transaction واحدة** (لا نصف حركة، و idempotent).

## حالات المستند
`Draft → Confirmed → Partially Processed → Completed`
إضافية: `Cancelled` (قبل أي حركة لاحقة)، `Reversed` (عُكس بمستند مقابل)،
`Returned` (مرتجع كامل)، `Partially Returned` (مرتجع جزئي).

## روابط الأرقام المقروءة (قاعدة P0 عامة)
- الرابط العام يستخدم **رقم المستند** لا الـ UUID:
  - `/erp/accounting/journal/JV-2026-0019`
  - `/erp/purchases/orders/PO-2026-0021` · `/erp/purchases/receipts/PR-2026-0014` · `/erp/purchases/invoices/PI-2026-0032`
  - `/erp/sales/orders/SO-2026-0045` · `/erp/sales/deliveries/DN-2026-0028` · `/erp/sales/invoices/SI-2026-0051`
- يبقى الـ UUID مفتاحًا داخليًا للعلاقات. `documentNumber` فريد داخل (الشركة + نوع المستند).
- تُحدّث الصفحات والروابط والبحث والتحويلات لاستخدام الرقم، مع تحويل الروابط القديمة (UUID) إلى الجديدة.

## القيد اليومي اليدوي
- الحفظ ينشئ قيد `Draft` قابلًا للتعديل/الحذف.
- زر **ترحيل القيد** يتحقق من توازن المدين=الدائن ثم يرحّل ويقفل.
- العكس ينشئ قيدًا جديدًا «عكس القيد رقم...» مربوطًا بالأصل. لا تتغير الأرصدة قبل الترحيل.

## دورة المشتريات
`أمر شراء → إذن استلام → فاتورة شراء → سند دفع` (+ مرتجع استلام، مرتجع فاتورة/إشعار مدين، استرداد/تسوية).

| المستند | عند الحفظ | عند التأكيد |
|---|---|---|
| أمر شراء | لا شيء | اعتماد الطلب فقط |
| إذن استلام | لا شيء | إدخال المخزون + قيد الاستلام |
| فاتورة شراء | لا شيء | إثبات المورد والضريبة وتسوية الاستلام |
| سند دفع | لا شيء | تخفيض رصيد المورد والخزينة/البنك |

**ضوابط:** كميات مستلمة من المتبقي في أمر الشراء، استلام جزئي + Backorder، منع تجاوز المطلوب إلا بصلاحية/سماح، الفاتورة من استلامات مؤكدة، مطابقة ثلاثية (PO↔GRN↔Invoice)، توزيع الدفع على فواتير، حالات مستقلة (استلام/فوترة/دفع).

## دورة المبيعات
`أمر بيع → إذن صرف → فاتورة بيع → سند قبض` (+ مرتجع صرف، مرتجع فاتورة/إشعار دائن، رد/تسوية).

| المستند | عند الحفظ | عند التأكيد |
|---|---|---|
| أمر بيع | لا شيء | اعتماد + حجز الكمية حسب السياسة |
| إذن صرف | لا شيء | إخراج المخزون + إثبات COGS |
| فاتورة بيع | لا شيء | إثبات العميل والإيراد والضريبة |
| سند قبض | لا شيء | تخفيض رصيد العميل + زيادة الخزينة/البنك |

**ضوابط:** تحديد المخزن يدويًا (لا أول مخزن تلقائيًا)، منع الصرف فوق المتاح إلا بصلاحية، صرف جزئي + Backorder، الفاتورة من المصروف فقط، تحصيل جزئي موزّع، فحص حد الائتمان قبل تأكيد أمر البيع، حالات مستقلة (حجز/صرف/فوترة/تحصيل).

## المرتجعات
مستند جديد (ليس تعديلًا): علامة **مرتجع** + `originalDocumentId`، لا يتجاوز الرصيد القابل للمرتجع، يعكس نفس المخزن/الصنف/التكلفة/الضرائب/الحسابات، جزئي أو كامل، يظهر داخل صفحة الأصل، لا مرتجع مرتين لنفس الكمية، الاسترداد بسند مستقل.

## المخزون
`Stock Ledger` مصدر الحقيقة. كل حركة: الصنف/المخزن/الموقع/الداخل/الخارج/التكلفة/القيمة/نوع ورقم المستند. هيكل `مخزن → منطقة → رف → صندوق`. عرض المتاح/المحجوز/الفعلي/المتوقع. التحويل والجرد والتسوية والرصيد الافتتاحي مستندات مستقلة `Draft → Confirmed`. **WAC** في الإصدار الأول. تقارير: حركة الصنف، الرصيد، التقييم، الأعمار، إعادة الطلب.

## صفحة المستند (قياسية لكل مستند)
رأس + أطراف + تواريخ + مخزن + عملة · جدول أصناف/كميات/أسعار/ضرائب · إجماليات/ملاحظات/مرفقات · شريط حالة · أزرار (حفظ، تأكيد، إلغاء، إنشاء المستند التالي، إنشاء مرتجع) · قسم **المستندات المرتبطة** · سجل تدقيق (من أنشأ/عدّل/أكد/ألغى/عكس).

## مراحل التنفيذ
- **P0 — محرك المستندات:** فصل الحفظ عن التأكيد، منع التحويل التلقائي، توحيد الحالات، الترقيم الآمن (atomic sequence)، روابط الأرقام المقروءة، الصلاحيات، سجل التدقيق.
- **P1 — المخزون:** الدفتر، الحجز، المخازن الهرمية، الاستلام/الصرف الجزئي، التحويل والجرد.
- **P1 — المشتريات:** أمر→استلام→فاتورة→دفع + مطابقة + مرتجعات.
- **P1 — المبيعات:** أمر→صرف→فاتورة→قبض + ائتمان + مرتجعات.
- **P2 — التقارير والجودة:** تقارير تشغيلية/محاسبية، اختبارات تكامل، أداء، إقفال مالي.

## معايير القبول
- حفظ أي مستند لا يغيّر أي رصيد.
- التأكيد وحده ينشئ الحركة؛ لا مستند لاحق دون إجراء صريح.
- لا تأكيد لمستند ناقص/غير متوازن؛ لا تكرار حركة عند التأكيد مرتين.
- مجموع القيود متوازن دائمًا؛ المرتجع مرتبط بالأصل ويعكسه بدقة.
- كل رصيد مخزون/حساب قابل للتتبع حتى المستند المصدر.

## فجوة الوضع الحالي مقابل الخطة (للترحيل في P0)
- **يطابق:** فاتورة بيع/شراء (Draft ثم ترحيل)، القيد اليدوي (Draft/Post/Reverse)، GRN/Delivery (تأكيد ذري + GRNI/COGS)، دفتر مخزون WAC، عكس القيود.
- **يحتاج تعديل:** أوامر البيع/الشراء تُنشأ `CONFIRMED` (يجب Draft ثم تأكيد) · سندات القبض/الصرف والمرتجعات تُرحّل فور الإنشاء (يجب Draft ثم تأكيد) · التحويل المخزني/التسوية تُنشأ مؤكدة · لا يوجد استلام/صرف/فوترة جزئية · لا تعديل لمسودة القيد · الروابط بالـ UUID لا بالرقم · سجل تدقيق لا يغطي حركات ERP · الترقيم read-last+1 (يحتاج atomic sequence).
